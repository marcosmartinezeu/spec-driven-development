# Skill — dashboard-ui

## Purpose

Design and implement a clean, usable, and consistent dashboard-style user interface using Tailwind CSS and Flowbite.

This skill ensures that the UI is not only visually consistent but also usable, accessible, and aligned with real user workflows.

The objective is clarity and usability, not visual complexity.

---

## Use This Skill When

Use this skill for:

* designing layouts (dashboard, sidebar, navbar)
* building forms and user flows
* creating Twig templates
* applying Tailwind CSS
* using Flowbite components
* improving UX
* ensuring responsive design
* improving accessibility
* creating reusable UI components

If the question is "how should this look and behave for the user?", this skill should be used.

---

## Core Rule

The UI must help the user complete tasks quickly and clearly.

Visual design must serve usability.

Avoid:

* unnecessary visual complexity
* unclear interactions
* inconsistent layouts

---

## Technology Rules

Use:

* Twig for templates
* Tailwind CSS for styling
* Flowbite for components
* minimal JavaScript (vanilla or Flowbite JS)

Avoid:

* introducing heavy frontend frameworks (React/Vue) at this stage
* custom CSS unless necessary

Prefer composition of existing components.

---

## Layout Rules

Dashboard layout should include:

* sidebar navigation
* top navigation bar
* main content area
* clear spacing and hierarchy

Consistency is mandatory.

---

## Form Design Rules

Forms must be:

* clear
* minimal
* validated
* with visible error messages
* with clear call-to-action buttons

Labels must always be present.

---

## UX Rules

Ensure:

* clear user flows
* predictable navigation
* feedback on actions (success/error)
* empty states handled
* loading states when necessary

The user must always know what is happening.

---

## Responsive Rules

The UI must work on:

* desktop
* tablet
* mobile

Use Tailwind responsive utilities.

Mobile usability is required, not optional.

---

## Accessibility Rules

Ensure:

* semantic HTML
* proper labels
* keyboard navigation
* readable contrast

Accessibility is part of quality.

---

## Reusability Rule

Create reusable components when:

* patterns repeat
* forms share structure
* layouts are reused

Avoid duplication in templates.

---

## Forbidden

* mixing business logic into templates
* complex JavaScript without justification
* inconsistent UI patterns
* hidden actions
* unclear navigation
* ignoring responsive behavior

---

## Expected Output

This skill should produce:

* clean dashboard UI
* consistent layouts
* usable forms
* responsive pages
* clear user flows
* maintainable Twig templates

The UI should feel simple, predictable, and professional.
